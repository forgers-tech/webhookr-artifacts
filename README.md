# webhookr-artifacts

Versioned, dated **artifact store** for marketing and product-communication work — briefs,
competitive analysis, landing-page specs, launch copy — mostly produced by the `/design` workflow.

The point is **history**: nothing is overwritten; every revision is a new dated, versioned file, so
we can see how positioning and messaging evolved.

- Convention and rules: [CLAUDE.md](CLAUDE.md)
- Scannable history: [marketing/INDEX.md](marketing/INDEX.md)

```
marketing/<YYYY-MM-DD>-<topic>/<YYYY-MM-DD>-<artifact>-v<N>.md
```
