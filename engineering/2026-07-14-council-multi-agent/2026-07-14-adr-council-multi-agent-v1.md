# ADR-006: `/council` runs as a multi-agent orchestrator, not a simulated committee

## Status

Approved (2026-07-14) — shipped in `forgers-tech/skills` v1.6.0 ([PR #24](https://github.com/forgers-tech/skills/pull/24)).

Extends [ADR-005](../2026-07-14-decision-ledger-sdlc/2026-07-14-adr-decision-ledger-sdlc-v1.md): the Decision-Ledger contract (Card Admission Criteria, Verified Story, `D-#` ledger) is preserved unchanged; this ADR changes *how* the Ledger is produced.

## Context

Under ADR-005, `/council` owns every engineering decision and is the hard entry gate of the feature lifecycle. But the command was still a **single model asked to simulate a committee**: one reasoning pass invented the specialists, their opinions, their disagreements, and the consensus between them. The output document was often excellent, yet the process was fundamentally linear — an *illusion* of independent reasoning rather than the real thing. Because a single pass produces every "voice," a genuinely independent objection (the kind that catches a bad decision) is only as likely as the one model happening to argue against itself convincingly. Conflicts, and therefore the negotiation that resolves them, were not real events; they were narrative.

The runtime now makes real independence cheap: Claude Code (and Codex, GPT CLI, or any comparable layer) can launch independent agent executions, each in its own isolated context. That capability makes it possible to replace the simulation with actual parallel analyses.

## Decision

Redesign `/council` as an **orchestration protocol**. The Chair stops reasoning as the committee and instead coordinates genuinely independent specialist agents.

- **Prime directive: orchestrate, never simulate.** The Chair authors no specialist output. Every understanding, concern, proposal, conflict, revision, and consensus originates from an actual independent execution (Claude Code **Task tool**, `subagent_type: general-purpose`; `Explore` for the Historian). If the runtime cannot launch independent agents, the council **blocks** — it must not fall back to a single-pass imitation.
- **Phased protocol (0–7):** Chair builds a Council Plan + Evidence Package → **Round 1**: one independent, stateless agent per selected specialist, launched in parallel, blind to each other → Chair aggregates into a **Consensus Matrix** → **conflict-resolution round**: conflicted specialists relaunched with the matrix + all Round-1 reports, positions may change with stated reasons → **Decision Candidates** → **challenge round**: real challenge→response agent pairs → **Council Decision** → the Scribe writes the Council Record. Coordination happens **only through artifacts**; agents share no hidden state.
- **Independence rules:** in Round 1 a specialist receives only the Council Plan, Evidence Package, repository, and card — never another specialist's output. Conflicts are named only where two real outputs diverge, and both are quoted. Where agents agree, that is stated plainly rather than dramatized.
- **Roster rationalized:** the generic "Staff Engineer" is removed; Platform+SRE merged; API Designer folded into Backend unless the change is contract-first. **Historian and Devil's Advocate become permanent independent agents** — precedent and dissent must come from real isolated executions, which is the core anti-simulation fix. The Chair selects Required / Optional / Not-required specialists per request and records why.
- **Portable by construction:** specialist prompt contracts live **inline** in the command (Mission / Inputs / Outputs / Authority / Forbidden / Evaluation / Success), launched as generic independent executions. Nothing depends on Claude internally playing multiple people, so the design is runtime-agnostic.

Shipped alongside this: every command's Execution Trace (ADR-005) gains a **Tokens** column, filled best-effort from the runtime's reported usage (summed across subagent executions for the council) and marked `n/a` when the runtime does not expose usage — never invented.

## Why this exists (rationale, not implementation)

- **Real independence beats simulated independence.** The value of a review board is that its members reason separately and can genuinely disagree. A single model cannot truly disagree with itself; isolated executions can. Conclusions now *emerge* from multiple independent analyses instead of being generated by one pass.
- **No fabricated debate.** Tying every conflict to two quotable agent outputs removes the temptation to manufacture drama or, worse, to manufacture a false consensus. If there is no disagreement, the record says so.
- **Determinism and traceability.** A fixed phase protocol, explicit specialist-selection rules, and artifact-only coordination make each run reproducible and auditable: you can see which agent said what, in which round, and why a position changed.
- **Preserve the contract, change the engine.** Downstream stages already depend on the Decision Ledger. Keeping the Ledger (and the admission gate and Verified Story) identical means the orchestration change is invisible to `/upstream`, `/gatekeeper`, `/downstream`, and `artifact-reconciliation` — only the *quality and honesty* of how decisions are reached improves.
- **Runtime-agnostic on purpose.** Inline contracts + generic independent executions mean the same protocol runs on Claude Code today and on any future multi-agent runtime, rather than binding the council to one vendor's agent-definition format.

## Consequences

- `/council` now requires a runtime capable of independent agent executions; without one it blocks by design. This is a deliberate cost — the council refuses to pretend.
- A council run is more expensive (multiple executions, multiple rounds) and slower than a single pass. That is the price of real independence; adaptive depth (lightweight / standard / critical) scales the number of agents and rounds to the decision so cheap decisions stay cheap.
- The implementation (command body, prompt contracts, phase protocol, packaging, versioning) lives in `forgers-tech/skills`; this ADR records only the **why**. They cross-link: see `commands/council.md`, `CLAUDE.md`, `README.md` in the `skills` repo.
